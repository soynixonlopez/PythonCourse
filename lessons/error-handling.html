üéØ What is Error Handling?

Error handling lets your program deal with errors gracefully instead of crashing!

üåç Real-Life Analogy

Like a safety net for trapeze artists:

  • try - Perform the trick
  • except - The safety net catches falls
  • finally - Bow to the audience regardless

üìã Without Error Handling

Python - CRASHES! 
age = int(input("Enter age: "))
# User enters "abc" ‚Üí ValueError: invalid literal
# Program crashes! üí•

üìã With Error Handling

Python - SAFE! 
try:
    age = int(input("Enter age: "))
except ValueError:
    print("Please enter a valid number!")
# Program continues! ‚úÖ

üîß try-except Block

üìù Basic Syntax

Python 
try:
    # Code that might cause an error
    result = 10 / 0
except:
    # Code to run if ANY error occurs
    print("An error occurred!")

üìù Catching Specific Exceptions

Python 
try:
    num = int(input("Enter number: "))
    result = 100 / num
    print(f"Result: {result}")
except ValueError:
    print("That's not a valid number!")
except ZeroDivisionError:
    print("Cannot divide by zero!")

üìù Getting Error Details

Python 
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"Error: {e}")
# Output: Error: division by zero

üìù Catching Multiple Exceptions

Python 
try:
    # risky code
    pass
except (ValueError, TypeError):
    print("Value or Type error!")
except Exception as e:
    print(f"Unexpected error: {e}")

üèÅ finally & else Clauses

üìù The else Clause

Runs only if NO exception occurred.

Python 
try:
    num = int(input("Enter number: "))
except ValueError:
    print("Invalid input!")
else:
    print(f"You entered: {num}")  # Only runs if no error

üìù The finally Clause

ALWAYS runs, error or not.

Python 
try:
    file = open("data.txt")
    data = file.read()
except FileNotFoundError:
    print("File not found!")
finally:
    print("Cleanup complete")  # ALWAYS runs

üìù Complete Structure

Python 
try:
    # Risky code
    result = 10 / 2
except ZeroDivisionError:
    # Handle specific error
    print("Can't divide by zero!")
except Exception as e:
    # Handle any other error
    print(f"Error: {e}")
else:
    # No error occurred
    print(f"Result: {result}")
finally:
    # Always runs
    print("Done!")

üí° When to Use finally?

  • Closing files
  • Closing database connections
  • Releasing resources
  • Cleanup operations
