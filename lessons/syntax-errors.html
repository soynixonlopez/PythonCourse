Learn to debug like a pro! üêõ

üìö How to Read Error Messages

Don't panic when you see an error! Python is actually trying to help you by telling you what went wrong.

üåç Real-Life Analogy

Error messages are like a GPS saying "You missed the turn! Go back 100 meters and turn left." They tell you where the problem is and what went wrong.

üîç Anatomy of an Error

Error Message 
Traceback (most recent call last):
  File "program.py", line 3, in <module>
    print(Hello)
          ^^^^^
NameError: name 'Hello' is not defined

üìã Understanding the Parts

PartWhat It Tells You
TracebackStart of error report
File "program.py"Which file has the error
line 3Which line number to check
print(Hello)The problematic code
^^^^^Points to the exact problem
NameErrorType of error
name 'Hello' is not definedExplanation of what's wrong

üí° Pro Tips for Debugging

  • Read from bottom to top - The error type and description are at the bottom
  • Check the line number - Go to that line in your code
  • Google the error - Copy the error message and search for solutions
  • Check the line BEFORE - Sometimes the error is on the previous line!

‚ùå SyntaxError

A SyntaxError means Python can't understand your code because you broke the "grammar rules."

üî¥ Common Causes

1. Missing Quotes

‚ùå Wrong

print(Hello World)

‚úÖ Correct

print("Hello World")

2. Missing Parentheses

‚ùå Wrong

print "Hello"

‚úÖ Correct

print("Hello")

3. Missing Colon (:)

‚ùå Wrong

if age >= 18
    print("Adult")

‚úÖ Correct

if age >= 18:
    print("Adult")

4. Mismatched Quotes

‚ùå Wrong

name = "Alex'
name = 'Hello"

‚úÖ Correct

name = "Alex"
name = 'Hello'

5. Using = Instead of ==

‚ùå Wrong

if x = 5:  # Assignment!
    print("x is 5")

‚úÖ Correct

if x == 5:  # Comparison!
    print("x is 5")

üìê IndentationError

Python uses indentation (spaces at the beginning of lines) to group code. If it's wrong, you get an IndentationError!

‚ö†Ô∏è The Rule

After if, for, while, def, etc., the next lines must be indented (usually 4 spaces).

üî¥ Common Causes

1. No Indentation After Colon

‚ùå Wrong

if True:
print("Hello")  # Not indented!

‚úÖ Correct

if True:
    print("Hello")  # 4 spaces

2. Inconsistent Indentation

‚ùå Wrong

if True:
    print("Line 1")
      print("Line 2")  # Too many spaces!

‚úÖ Correct

if True:
    print("Line 1")
    print("Line 2")  # Same indent

3. Mixing Tabs and Spaces

The Problem 
# Sometimes code LOOKS right but uses different
# invisible characters (tabs vs spaces)
if True:
‚Üí   print("uses tab")
    print("uses spaces")  # Inconsistent!

üí° How to Fix

  • Configure your editor to use 4 spaces for indentation
  • Enable "Show whitespace" in your editor
  • Use the "Convert indentation to spaces" feature

üî§ NameError

A NameError means Python doesn't recognize a name you used. Usually it's a typo or undefined variable!

üî¥ Common Causes

1. Typos in Variable Names

‚ùå Wrong

username = "Alex"
print(usernme)  # Typo!

‚úÖ Correct

username = "Alex"
print(username)

2. Using Before Defining

‚ùå Wrong

print(score)  # Used before defined!
score = 100

‚úÖ Correct

score = 100   # Define first
print(score)  # Then use

3. Forgetting Quotes on Strings

‚ùå Wrong

print(Hello)  # Python thinks Hello
              # is a variable!

‚úÖ Correct

print("Hello")  # Quotes make it text

4. Case Sensitivity

‚ùå Wrong

Name = "Alex"
print(name)  # Different case!

‚úÖ Correct

name = "Alex"
print(name)  # Same case

üî¢ TypeError

A TypeError means you're trying to do something with the wrong type of data!

üî¥ Common Causes

1. Adding String + Number

‚ùå Wrong

age = "15"
next_year = age + 1  # Can't add!

‚úÖ Correct

age = int("15")  # Convert first
next_year = age + 1

2. Concatenating Without str()

‚ùå Wrong

score = 100
print("Score: " + score)

‚úÖ Correct

score = 100
print("Score: " + str(score))
# Or use f-string:
print(f"Score: {score}")

3. Calling Non-Callable

‚ùå Wrong

message = "Hello"
message()  # String is not a function!

‚úÖ Correct

message = "Hello"
print(message)  # Just use it directly
