üéØ What are Functions?

A function is a reusable block of code that performs a specific task. Instead of writing the same code over and over, you write it once and call it whenever needed!

üåç Real-Life Analogy

Think of a function like a recipe:

  • Has a name (e.g., "make_pizza")
  • Takes ingredients (parameters)
  • Follows steps (the code)
  • Returns a result (the pizza!)

üìã Why Use Functions?

BenefitDescription
ReusabilityWrite once, use many times
OrganizationBreak code into logical pieces
ReadabilityGive meaningful names to operations
MaintainabilityFix bugs in one place
TestingTest individual pieces of code

üí° Built-in Functions You Already Know!

print(), len(), input(), range(), type() - all functions!

üîß Defining Functions

üìù Basic Syntax

Python 
def function_name():
    """Docstring: describes what the function does"""
    # Function body (code)
    print("Hello from the function!")

üéØ Simple Function Example

Python 
# Define the function
def greet():
    """Print a greeting message"""
    print("Hello, World!")
    print("Welcome to Python!")

# Call the function
greet()  # Output: Hello, World! Welcome to Python!

# Call it again!
greet()  # Runs the same code again

‚ö†Ô∏è Important Rules!

  • Function name must start with a letter or underscore
  • Use lowercase with underscores (snake_case)
  • Don't forget the colon : after def
  • Indent the function body (4 spaces)
  • Call with parentheses: greet() not greet

üì• Function Parameters

Parameters let you pass data INTO your function.

üìù Function with Parameters

Python 
def greet(name):
    """Greet a person by name"""
    print(f"Hello, {name}!")

# Call with argument
greet("Alice")   # Hello, Alice!
greet("Bob")     # Hello, Bob!

üìù Multiple Parameters

Python 
def add_numbers(a, b):
    """Add two numbers and print result"""
    result = a + b
    print(f"{a} + {b} = {result}")

add_numbers(5, 3)    # 5 + 3 = 8
add_numbers(10, 20)  # 10 + 20 = 30

üìù Default Parameters

Python 
def greet(name, greeting="Hello"):
    """Greet with optional greeting"""
    print(f"{greeting}, {name}!")

greet("Alice")              # Hello, Alice!
greet("Bob", "Hi")          # Hi, Bob!
greet("Charlie", "Welcome") # Welcome, Charlie!

üìù Keyword Arguments

Python 
def describe_pet(name, animal="dog"):
    print(f"{name} is a {animal}")

# Using keyword arguments
describe_pet(name="Buddy")                  # Buddy is a dog
describe_pet(name="Whiskers", animal="cat") # Whiskers is a cat
describe_pet(animal="bird", name="Tweety")  # Order doesn't matter!

üí° Parameter vs Argument

  • Parameter: Variable in function definition
  • Argument: Value passed when calling

üì§ Return Values

Use return to send data back from the function.

üìù Basic Return

Python 
def add(a, b):
    """Return the sum of a and b"""
    return a + b

# Store the result
result = add(5, 3)
print(result)  # 8

# Use directly
print(add(10, 20))  # 30

üìù Return Multiple Values

Python 
def get_stats(numbers):
    """Return min, max, and sum of a list"""
    return min(numbers), max(numbers), sum(numbers)

minimum, maximum, total = get_stats([1, 5, 3, 9, 2])
print(f"Min: {minimum}, Max: {maximum}, Sum: {total}")
# Min: 1, Max: 9, Sum: 20

üìù Return Ends the Function

Python 
def is_positive(num):
    if num > 0:
        return True
    return False

print(is_positive(5))   # True
print(is_positive(-3))  # False

‚ö†Ô∏è No Return = None

Functions without return (or just return alone) return None

üîç Variable Scope

Scope determines where a variable can be accessed.

üìù Local vs Global Scope

Python 
global_var = "I'm global!"  # Global scope

def my_function():
    local_var = "I'm local!"  # Local scope
    print(global_var)  # Can access global
    print(local_var)   # Can access local

my_function()
print(global_var)  # Works
# print(local_var)  # ‚ùå NameError! local_var not defined here

üìù The global Keyword

Python 
counter = 0

def increment():
    global counter  # Use the global variable
    counter += 1

increment()
increment()
print(counter)  # 2

üí° Best Practice

Avoid global when possible. Pass values as parameters and use return values instead!
