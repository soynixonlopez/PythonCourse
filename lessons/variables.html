üì¶ What are Variables?

A variable is like a labeled box that stores information. You give it a name, and Python remembers what's inside!

üåç Real-Life Analogy

Imagine you have labeled boxes in your room:

  • üì¶ Box labeled "my_age" contains: 15
  • üì¶ Box labeled "my_name" contains: "Alex"
  • üì¶ Box labeled "is_student" contains: True

Variables work the same way - they're containers with names!

üéØ Why Do We Need Variables?

Without Variables With Variables
Type the same value over and over Store once, use many times
Hard to change values Change in one place, updates everywhere
Code is confusing Code is clear and readable

üìù Basic Example

Python 
# Creating variables
name = "Python Student"
age = 15
score = 95.5

# Using variables
print(name)    # Output: Python Student
print(age)     # Output: 15
print(score)   # Output: 95.5

‚ú® Creating Variables

Creating a variable in Python is super easy - just use the = sign!

üìù Syntax

üéØ Examples

üåü Best Practices (Conventions)

üêç snake_case - The Python Way!

Python uses snake_case for variable and function names. This means:

  • All letters are lowercase
  • Words are separated by underscores (_)
  • This is defined in PEP 8 (Python's official style guide)

üìã Python Naming Conventions (PEP 8)

Type Convention Example
Variables snake_case user_name, total_score
Functions snake_case calculate_total(), get_user()
Constants SCREAMING_SNAKE_CASE MAX_SIZE, PI
Classes PascalCase MyClass, StudentRecord
Python 
# ‚úÖ Good: Use snake_case (the Python way!)
user_name = "Alex"
total_score = 100
is_active = True
first_name = "John"
last_login_date = "2024-01-15"

# ‚ùå Avoid: camelCase (used in JavaScript, Java)
userName = "Alex"      # Works, but NOT Pythonic!
totalScore = 100       # This is not Python style

# ‚ùå Avoid: PascalCase for variables (use for classes only)
UserName = "Alex"      # Looks like a class name!

# ‚úÖ Good: Descriptive names tell you what the variable stores
student_count = 25
max_temperature = 100
is_logged_in = True

# ‚ùå Avoid: Single letters (except in loops or math)
x = 25                 # What does x mean?
n = 100                # Confusing!
# Exception: for i in range(10) is okay!

üîÑ Other Languages Use Different Styles

  • JavaScript/Java: camelCase ‚Üí userName
  • Python: snake_case ‚Üí user_name
  • C#: PascalCase ‚Üí UserName

When writing Python, always use snake_case to follow the community standards!

üîí Constants

A constant is a variable whose value should never change. Python doesn't enforce this, but we use a naming convention to show it.

üìù Convention: ALL_CAPS

Python 
# Constants use ALL_CAPS with underscores
PI = 3.14159
MAX_SPEED = 120
GRAVITY = 9.8
WEBSITE_URL = "https://python.org"

# Regular variables use lowercase
radius = 5
speed = 60

# Using constants
circumference = 2 * PI * radius
print(circumference)  # 31.4159

üéØ When to Use Constants?

  • Mathematical values - PI, E, GOLDEN_RATIO
  • Configuration - MAX_USERS, DEFAULT_TIMEOUT
  • Fixed values - DAYS_IN_WEEK, MONTHS_IN_YEAR
  • URLs and paths - API_URL, DATABASE_PATH

üí° Important Note

Python doesn't actually prevent you from changing constants - it's just a convention. The ALL_CAPS tells other programmers "don't change this!"

PI = 3.14159
PI = 999      # This works, but DON'T do it!
