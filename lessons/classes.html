üéØ What is OOP?

Object-Oriented Programming (OOP) is a way to structure code by creating "objects" that contain both data and behavior.

üåç Real-Life Analogy

Think of a Class as a blueprint for a house:

  • Class = Blueprint (design)
  • Object = Actual house (instance)
  • Attributes = House features (color, rooms)
  • Methods = What house can do (open door)

üìã Key Concepts

TermDescription
ClassBlueprint/template for objects
ObjectInstance of a class
AttributeData stored in object
MethodFunction inside a class
__init__Constructor (runs when creating object)
selfReference to current object

üîß Creating Classes

üìù Basic Class Syntax

Python 
class Dog:
    """A simple Dog class"""
    
    def __init__(self, name, age):
        """Initialize dog with name and age"""
        self.name = name
        self.age = age
    
    def bark(self):
        """Make the dog bark"""
        print(f"{self.name} says: Woof!")

üìù Creating Objects (Instances)

Python 
# Create dog objects
my_dog = Dog("Buddy", 3)
your_dog = Dog("Max", 5)

# Access attributes
print(my_dog.name)   # Buddy
print(your_dog.age)  # 5

# Call methods
my_dog.bark()   # Buddy says: Woof!
your_dog.bark() # Max says: Woof!

üí° About self

self refers to the current instance. It must be the first parameter of every method, but you don't pass it when calling!

‚ö° Methods

üìù Instance Methods

Python 
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance
    
    def deposit(self, amount):
        self.balance += amount
        print(f"Deposited ${amount}. Balance: ${self.balance}")
    
    def withdraw(self, amount):
        if amount <= self.balance:
            self.balance -= amount
            print(f"Withdrew ${amount}. Balance: ${self.balance}")
        else:
            print("Insufficient funds!")
    
    def get_balance(self):
        return self.balance

# Using the class
account = BankAccount("Alice", 100)
account.deposit(50)    # Deposited $50. Balance: $150
account.withdraw(30)   # Withdrew $30. Balance: $120

üìù Methods Can Return Values

Python 
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)

rect = Rectangle(5, 3)
print(rect.area())       # 15
print(rect.perimeter())  # 16

üì¶ Attributes

üìù Instance vs Class Attributes

Python 
class Dog:
    # Class attribute (shared by ALL dogs)
    species = "Canis familiaris"
    
    def __init__(self, name):
        # Instance attribute (unique to each dog)
        self.name = name

dog1 = Dog("Buddy")
dog2 = Dog("Max")

print(dog1.name)      # Buddy (unique)
print(dog2.name)      # Max (unique)
print(dog1.species)   # Canis familiaris (shared)
print(dog2.species)   # Canis familiaris (shared)

üìù Modifying Attributes

Python 
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def birthday(self):
        self.age += 1
        print(f"Happy Birthday! Now {self.age} years old.")

person = Person("Alice", 25)
print(person.age)    # 25
person.birthday()   # Happy Birthday! Now 26 years old.
print(person.age)    # 26

üí° OOP Benefits

  • Encapsulation: Bundle data with methods
  • Reusability: Create many objects from one class
  • Organization: Cleaner, structured code
  • Real-world modeling: Objects mirror reality
