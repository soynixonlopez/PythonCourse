Store data as key-value pairs! üìñ

üéØ What are Dictionaries?

A dictionary stores data as key-value pairs. Instead of using indexes (0, 1, 2...), you use meaningful keys to access data!

üåç Real-Life Analogy

Like a real dictionary:

  • Word (key) ‚Üí Definition (value)
  • "Python" ‚Üí "A programming language"
  • Look up by the word, get the definition!

üìù Creating Dictionaries

Python 
# Empty dictionary
empty = {}

# Dictionary with data
person = {
    "name": "Alice",
    "age": 25,
    "city": "New York"
}

# Using dict() constructor
person2 = dict(name="Bob", age=30)

# Keys can be numbers or tuples (immutable types)
scores = {1: "First", 2: "Second"}

üìã Dictionary Properties

Property Description
Ordered Items maintain insertion order (Python 3.7+)
Mutable Can add, change, remove items
No Duplicate Keys Each key must be unique
Keys Must Be Immutable Strings, numbers, tuples (not lists)

üîë Accessing Dictionary Values

üìù Using Square Brackets

Python 
person = {"name": "Alice", "age": 25}

print(person["name"])  # "Alice"
print(person["age"])   # 25

# KeyError if key doesn't exist!
# print(person["email"])  # ‚ùå KeyError

üìù Using get() Method (Safer!)

Python 
person = {"name": "Alice", "age": 25}

# get() returns None if key doesn't exist
print(person.get("name"))     # "Alice"
print(person.get("email"))    # None (no error!)

# Provide default value
print(person.get("email", "N/A"))  # "N/A"

üìù Checking if Key Exists

Python 
person = {"name": "Alice", "age": 25}

if "name" in person:
    print("Name exists!")

if "email" not in person:
    print("No email found")

üí° Best Practice

Use get() when you're not sure if a key exists. Use [] when you're certain or want an error if missing.

‚úèÔ∏è Modifying Dictionaries

üìù Adding & Updating Items

Python 
person = {"name": "Alice"}

# Add new key-value pair
person["age"] = 25
person["city"] = "NYC"

# Update existing value
person["age"] = 26

# Update multiple with update()
person.update({"age": 27, "job": "Developer"})

üìù Removing Items

Python 
person = {"name": "Alice", "age": 25, "city": "NYC"}

# pop() - remove and return value
age = person.pop("age")
print(age)  # 25

# del - delete key
del person["city"]

# popitem() - remove last item
last = person.popitem()

# clear() - remove all items
person.clear()  # {}

üõ†Ô∏è Dictionary Methods

üìã All Dictionary Methods

Method Description
keys()Returns all keys
values()Returns all values
items()Returns all key-value pairs as tuples
get(key)Returns value (None if not found)
pop(key)Remove and return value
update(dict)Update with another dictionary
clear()Remove all items
copy()Returns shallow copy

üìù Keys, Values, Items

Python 
person = {"name": "Alice", "age": 25}

print(person.keys())    # dict_keys(['name', 'age'])
print(person.values())  # dict_values(['Alice', 25])
print(person.items())   # dict_items([('name', 'Alice'), ('age', 25)])

# Convert to list
keys_list = list(person.keys())

üîÑ Looping Through Dictionaries

üìù Different Ways to Loop

Python 
person = {"name": "Alice", "age": 25, "city": "NYC"}

# Loop through keys (default)
for key in person:
    print(key)  # name, age, city

# Loop through values
for value in person.values():
    print(value)  # Alice, 25, NYC

# Loop through both (most useful!)
for key, value in person.items():
    print(f"{key}: {value}")

üìù Nested Dictionaries

Python 
students = {
    "alice": {"age": 20, "grade": "A"},
    "bob": {"age": 21, "grade": "B"}
}

# Access nested value
print(students["alice"]["grade"])  # "A"

# Loop nested
for name, info in students.items():
    print(f"{name}: {info['grade']}")
